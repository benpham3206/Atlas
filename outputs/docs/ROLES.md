# Matrix Roles → Atlas Artifacts

Roles are operating metaphors, not chat personas. Apply **the Algorithm**: if a role needs a new subsystem, delete the subsystem and map to an existing artifact first.

| Matrix role | Owns | Atlas artifact |
| --- | --- | --- |
| **Owner** | Direction, irreversible approvals | Human review inbox, Board pause, policy owner |
| **Lead** | Taste, memory, domain continuity | GoalContract, Carbon Copy, ontology/domain pack |
| **Worker** | Scoped execution + proof | Tool Router + delegation + host agent (Hermes, Cursor, Codex) |
| **System** | Cadence, recovery, gates | `dev:personal`, bootstrap, cron heartbeat, `audit/verify` |

## UI surfaces

| Role | View |
| --- | --- |
| Owner | `?view=board`, `?view=review-inbox` |
| Lead | `?view=carbon-copy`, `?view=goal-contracts` |
| Worker | `?view=next-action`, MCP tools |
| System | `outputs/internal/NEXT_ACTION.md`, smoke gates |

## Proof bar

A role handoff is real when linked to audit evidence — not when copy exists alone.
